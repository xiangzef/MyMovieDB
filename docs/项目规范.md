# 项目目录规范

> **版本**: v1.0
> **制定日期**: 2026-04-27
> **目的**: 规范项目文件结构，避免文档和脚本散落主目录

---

## 目录结构规范

| 路径 | 用途 | 说明 |
|------|------|------|
| `docs/` | 文档文件 | 所有 .md 文档放在此目录 |
| `tests/` | 测试脚本 | 所有测试脚本放在此目录 |
| `backend/` | 后端代码 | Python 源码 |
| `frontend/` | 前端代码 | Vue/JS 静态文件 |
| `data/` | 数据文件 | SQLite DB、头像、封面等 |
| `dist/` | 构建产物 | 编译打包输出 |

**禁止在主目录放置文档文件或测试脚本。**

---

## 规则

### 1. 测试脚本 → `tests/`

- 所有测试脚本（Python/JS/HTML）放在 `tests/` 目录
- 命名规范: `test_*.py` 或 `check_*.py`
- 不允许在主目录或 `backend/` 下放置测试脚本

### 2. 文档文件 → `docs/`

- 所有 .md 文档放在 `docs/` 目录
- 禁止在主目录或 `backend/` 下放置 .md 文档
- 已归档: `CLAUDE.md`, `PROJECT_PLAN.md`, `OPTIMIZATION_PLAN.md`, `DEFECT_LOG.md`, `技术手册.md`

### 3. README.md 位置

- 项目根目录的 `README.md` 保留，用于展示项目概览
- `docs/README.md` 用于文档目录说明
