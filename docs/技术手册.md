## 1. 项目概述

**MyMovieDB** 是一个轻量级的本地影视库管理工具，支持影片信息自动刮削、Jellyfin 影视库导入、本地视频目录管理和智能搜索。

### 核心功能
- 🎬 影片刮削 - 从多个数据源自动获取影片信息
- 📁 Jellyfin 导入 - 一键扫描 Jellyfin 格式影视库
- 📂 本地视频管理 - 扫描本地目录，批量刮削视频文件
- 🔍 智能搜索 - 支持编号、标题搜索
- 🖼️ 封面管理 - 自动下载和本地存储封面图
- 🌐 Web 界面 - 响应式设计，支持移动端
- 📦 一键打包 - 可编译成 exe，系统托盘运行

## 5. 数据模型

### Pydantic 模型（models.py）

```python
# 影片基础模型
class MovieBase(BaseModel):
    code: str
    title: str
    title_cn: Optional[str] = None
    title_jp: Optional[str] = None
    release_date: Optional[str] = None
    duration: Optional[int] = None
    studio: Optional[str] = None
    maker: Optional[str] = None
    director: Optional[str] = None
    cover_url: Optional[str] = None
    preview_url: Optional[str] = None
    genres: Optional[List[str]] = None
    actors: Optional[List[str]] = None
    actors_male: Optional[List[str]] = None

# 影片响应模型
class MovieResponse(MovieBase):
    id: int
    created_at: datetime
    updated_at: datetime
    local_video_id: Optional[int] = None
    fanart_path: Optional[str] = None
    poster_path: Optional[str] = None
    thumb_path: Optional[str] = None
    local_video_path: Optional[str] = None
    source: Optional[str] = None
    source_type: Optional[str] = None
    video_path: Optional[str] = None
    scrape_status: Optional[str] = None
    local_videos: Optional[List[LocalVideoItem]] = None

# 本地视频项模型
class LocalVideoItem(BaseModel):
    id: int
    path: str
    size: Optional[int] = None
    duration: Optional[int] = None
    codec: Optional[str] = None
    is_scraped: Optional[int] = None

# 刮削请求模型
class ScrapeRequest(BaseModel):
    keyword: str
    save_cover: bool = True

# 刮削响应模型
class ScrapeResponse(BaseModel):
    success: bool
    message: str
    movie: Optional[MovieResponse] = None

# 影片列表响应模型
class MovieListResponse(BaseModel):
    total: int
    page: int
    page_size: int
    items: List[MovieResponse]
```

## 9. 核心流程

### 9.1 影片刮削流程
```
1. 用户输入番号
2. 后端解析番号
3. 依次尝试各数据源（优先级）
4. 合并各数据源信息
5. 翻译日文内容
6. 下载封面图
7. 裁剪生成 poster/thumb
8. 生成 NFO 文件
9. 保存到数据库
10. 返回结果
```

### 9.2 Jellyfin 导入流程
```
1. 用户输入影视库路径
2. 递归扫描目录结构
3. 提取番号（目录名）
4. 识别视频文件（mp4/avi/mkv/wmv）
5. 识别图片文件（poster/fanart/thumb）
6. 解析 NFO 文件（如有）
7. 写入数据库（标记 source=jellyfin）
8. 实时推送进度（SSE）
```

### 9.3 本地视频刮削流程
```
1. 扫描已添加的目录
2. 递归查找视频文件
3. 提取番号（从文件名）
4. 过滤无效番号
5. 写入 local_videos 表
6. 批量刮削（可选择）
7. 关联到 movies 表
8. 实时推送进度（SSE）
```

### 9.4 刮削完整性检查流程
```
1. 遍历所有影片
2. 检查必填字段（title, release_date, maker, actors）
3. 检查封面文件（poster/fanart/thumb）
4. 检查 NFO 文件
5. 标记问题项
6. 修复时重新刮削缺失项
```

## 10. 关键技术点

### 10.1 号码识别
- 标准正则: `[A-Z]{2,6}-\d{2,5}`
- 排除前缀: WEBIPZZ, HDABC, 390JNT, 1080PXYZ
- 特殊支持: FC2-PPV, HEYDOUGA, ABW 系列
- 自动标准化: JNT-001 → JNT-1

### 10.2 封面处理
- 存储结构: `data/covers/{番号}/{番号}-poster.jpg`
- poster 生成: 从 fanart 右半边中间裁剪
- thumb 生成: 从 poster 缩放
- NFO 生成: Kodi/Plex/Emby 兼容格式

### 10.3 并发控制
- 批量刮削使用 `threading.Lock`
- 停止标志使用字典 `_scrape_stop_flags`
- SSE 使用 `StreamingResponse`

### 10.4 错误处理
- Pydantic 验证失败返回 500（带详细信息）
- 数据库操作使用 try-except 保护
- 脏数据不影响其他记录

### 10.5 数据关联
- 一对多: movies → local_videos（通过 code）
- 一对一: movies.local_video_id → local_videos.id
- 封面路径覆盖策略: 刮削优先于本地

### 10.6 Jellyfin 兼容
- source_type = "jellyfin" 标记来源
- 封面检查跳过本地文件存在性验证
- 支持 NFO 解析和手动封面
